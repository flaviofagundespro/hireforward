# HireForward — Improvement Plan: Subscription \& Monetization Sprint

**Context:** The core product flow is working. This sprint closes the monetization and conversion loop by adding subscription management for both admins and subscribers, and a proper pricing/marketing section on the homepage with the "2026 professional" positioning.

***

## BLOCK A — ADMIN PANEL: Subscriber Management

### A.1 Subscribers Overview

Route: `/admin/subscribers`

- Summary cards: Total Active Subscribers | MRR (USD) | New This Month | Churned This Month
- Table: Company Name | Plan | Status | Started On | Next Renewal | Monthly Value | Actions
- Filters: by plan (Trial / Starter / Growth / Enterprise), status (active / past due / cancelled / trial)
- Actions per row: View Details | Change Plan | Suspend | Cancel Subscription
- Export CSV button


### A.2 Subscriber Detail

Route: `/admin/subscribers/[id]`

- Current plan badge + renewal date
- Payment history table: Date | Invoice | Amount | Status (Paid / Failed / Refunded)
- Usage vs. plan limits: active processes used / limit, candidates this month / limit, tokens this month / limit (progress bars)
- Manual override: "Extend trial", "Apply discount", "Force plan change"
- Button: "Send billing email"


### A.3 MRR \& Revenue Dashboard

Route: `/admin/revenue`

- Line chart: MRR evolution over last 12 months
- Cards: MRR | ARR | Average Revenue Per Account (ARPA) | Churn Rate %
- Table: Top 10 accounts by MRR
- Upcoming renewals (next 7 days) list

***

## BLOCK B — SUBSCRIBER SIDE: Subscription Self-Service

### B.1 Subscription Status Page

Route: `/settings/subscription`

- Current plan name + badge (Trial / Starter / Growth / Enterprise)
- Renewal date or "Trial ends on [date]"
- Usage meters:
    - Active processes: X / limit (progress bar)
    - Candidates evaluated this month: X / limit (progress bar)
    - AI tokens used this month: X / limit (progress bar)
- Billing history table: Date | Description | Amount | Invoice PDF link
- "Manage Payment Method" button (Stripe portal or placeholder)
- "Cancel Subscription" link (with confirmation modal)


### B.2 Upgrade Plan Button \& Flow

- Persistent banner/badge in the main dashboard sidebar when on Trial or when usage reaches 80% of limit: **"You've used 80% of your plan — Upgrade now →"**
- Upgrade modal triggered from banner or from `/settings/subscription`:
    - Shows all plan cards side by side with current plan highlighted
    - One-click "Upgrade to [Plan]" button per card
    - For MVP without Stripe: clicking "Upgrade" sends an email notification to the owner and shows "Your request has been received. We'll activate your plan within 24h."
    - With Stripe: redirect to Stripe Checkout


### B.3 Usage Alerts (client-facing)

- In-app toast/banner when usage hits 80% of any limit: processes, candidates, or tokens
- Email alert at 80% and 100%: "You've reached your plan limit. Upgrade to continue."
- After limit is reached: new processes/candidate invites are blocked with a clear upgrade CTA, not a generic error

## DATABASE CHANGES NEEDED

```sql
-- Add to companies table
ALTER TABLE companies ADD COLUMN plan VARCHAR(50) DEFAULT 'trial';
ALTER TABLE companies ADD COLUMN plan_started_at TIMESTAMP;
ALTER TABLE companies ADD COLUMN plan_renewal_at TIMESTAMP;
ALTER TABLE companies ADD COLUMN stripe_customer_id VARCHAR(100);
ALTER TABLE companies ADD COLUMN stripe_subscription_id VARCHAR(100);

-- New table: plan definitions
CREATE TABLE plans (
  id UUID PRIMARY KEY,
  name VARCHAR(50),           -- trial, starter, growth, enterprise
  price_usd DECIMAL(10,2),
  max_active_processes INT,
  max_candidates_per_month INT,
  max_tokens_per_month BIGINT,
  features JSONB,             -- {pdf_export, manager_view, custom_branding, priority_support}
  created_at TIMESTAMP DEFAULT NOW()
);

-- New table: billing events
CREATE TABLE billing_events (
  id UUID PRIMARY KEY,
  company_id UUID REFERENCES companies(id),
  event_type VARCHAR(50),     -- payment_success, payment_failed, plan_changed, trial_started
  amount_usd DECIMAL(10,2),
  stripe_invoice_id VARCHAR(100),
  description TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
```
