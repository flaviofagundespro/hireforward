# 🏗️ Architecture — HireForward

This document outlines the high-level architecture of HireForward, an AI-Native Recruitment SaaS. The platform is designed as a multi-tenant application that leverages a specialized Multi-Agent Pipeline to automate and enhance the hiring process.

## 📂 Monorepo Structure

The project uses **pnpm workspaces** for a clean separation of concerns:

-   `artifacts/hireforward`: React 19 Frontend.
-   `artifacts/api-server`: Express 5 Backend.
-   `lib/db`: Drizzle ORM schemas and database logic (PostgreSQL).
-   `lib/api-spec`: OpenAPI specifications and Orval configuration.
-   `lib/integrations-*`: Shared logic for third-party services (Anthropic, Clerk, Resend).

---

## 🖼️ The 4 Panel Architecture

The SaaS is conceptually divided into four functional panels:

### 1. Landing Page / Institutional
-   **Purpose**: Public face of the product, conversion, and information.
-   **Routes**: `/` (Home), `/sign-in`, `/sign-up`.
-   **Tech**: React Components with SEO-optimized metadata in `index.html`.
-   **Focus**: Pricing, FAQ, and "Why HireForward?" (solving the 2015 criteria vs 2026 skills gap).

### 2. Admin Panel (Super Administrator)
-   **Purpose**: Platform governance and multi-tenant management.
-   **Routes**: `/admin`, `/admin/companies/:id`.
-   **Features**:
    -   **Overview**: Global platform stats.
    -   **Tenant Management**: Activating/Suspending company accounts.
    -   **Cost Tracking**: Token usage and billing logs per company.
    -   **System Config**: Maintenance mode and AI provider settings.
-   **Auth**: Restricted to users with specific "admin" metadata via Clerk.

### 3. Company Panel (Tenant / HR)
-   **Purpose**: Where HR teams manage their recruitment cycles.
-   **Routes**: `/dashboard`, `/processes/*`, `/settings`.
-   **Features**:
    -   **Job Processes**: CRUD for job openings.
    -   **Kanban Board**: Drag-and-drop management of candidates.
    -   **Evaluation Viewer**: Visualization of AI-generated reports (Agent 3).
    -   **AI Configurator**: Guided setup for interview criteria (Agent 1).

### 4. Candidate Panel (Interview UI)
-   **Purpose**: Secure, mobile-first interface for candidates.
-   **Routes**: `/i/:token` (Short-lived secure links).
-   **Features**:
    -   **Adaptive Interview**: Real-time chat with Agent 2.
    -   **Streaming (SSE)**: Immediate AI feedback for a premium experience.
    -   **Zero Friction**: No login required; access is validated via JWT in the URL.

## 🤖 The Multi-Agent Pipeline

The core value proposition of HireForward is driven by three specialized AI agents:

### 1. Configurator Agent (HR → AI)
-   **Backend**: `routes/processes/index.ts`
-   **Role**: Acts as a consultant to HR.
-   **Input**: Role requirements, seniority, and area.
-   **Output**: A structured "Configuration Contract" containing the `agentSystemPrompt` for the interviewer and an `evaluationCriteria` rubric with weights.

### 2. Interviewer Agent (Candidate → AI)
-   **Backend**: `routes/interview/index.ts`
-   **Role**: Conducts the interview.
-   **Behavior**: Adaptive questioning, grounded in the prompt generated by Agent 1. It allows tools/internet usage, focusing on problem-solving reasoning rather than rote memorization.
-   **Output**: A detailed transcript with timestamps.

### 3. Evaluator Agent (Transcript → Report)
-   **Backend**: `triggerEvaluation` function.
-   **Role**: Independent analyst.
-   **Input**: The full transcript + the rubric from Agent 1.
-   **Output**: A structured JSON report including:
    -   **Overall Score** (0-100).
    -   **Criterion Scores** (1-5 with evidence-based justification).
    -   **Highlights & Red Flags**.
    -   **Recommendation** (Strong Hire, Hire, On the Fence, No Hire).

## 🔐 Security & Multi-tenancy

-   **Data Isolation**: Every table in `lib/db` includes a `company_id` column.
-   **Authentication**: Handled by Clerk (JWT/OAuth) for HR/Admins.
-   **Authorization**: Backend middlewares enforce that users can only access resources belonging to their `companyId`.
-   **Candidate Access**: Secured by JWT invite tokens signed with a server-side secret.
