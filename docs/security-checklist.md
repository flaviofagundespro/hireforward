# 🔐 Security Checklist — HireForward

This document outlines the security validations performed on the HireForward codebase and identifies areas that require attention before production deployment.

## ✅ Validated (Security Passed)

### 💳 Stripe & Billing
- [x] **No Hardcoded Secrets**: Stripe `secret_key` and `webhook_secret` are fetched dynamically via Replit Connectors API. No `sk_` keys in the codebase.
- [x] **Webhook Signature Verification**: Handled via `StripeSync.processWebhook`. Requests without a valid `stripe-signature` are rejected (400 Bad Request).
- [x] **Raw Body Handling**: The webhook route is registered before `express.json()` using `express.raw()` to ensure cryptographic signature validation works.
- [x] **Authenticated Billing Endpoints**: Checkout sessions and Customer Portal links can only be generated by authenticated HR users for their specific `companyId`.

### 🛡️ Authentication & Authorization
- [x] **Tenant Isolation**: Every database query in the API server includes a `WHERE company_id = ...` clause. No cross-tenant data exposure detected.
- [x] **JWT Candidate Tokens**: Interview access is secured via JWT tokens with a configurable expiry. Tokens are signed with `SESSION_SECRET`.
- [x] **Clerk Integration**: Production-ready auth provider for HR users.

### 📂 File Hygiene
- [x] **.gitignore**: Correctly configured to exclude `.env`, `node_modules`, `.cache`, and Replit metadata.
- [x] **Public Assets**: Shared reports (`/report/:token`) have a hardcoded 30-day expiry check in the backend logic.

---

## ⚠️ Attention Required (Pre-Production)

### 1. Domain & SSL
- [ ] **Custom Domain**: Ensure `hireforward.ai` is connected via Replit's "Custom Domains" to get managed SSL/TLS.
- [ ] **HSTS**: Consider adding Strict-Transport-Security headers once the custom domain is live.

### 2. Environment Secrets
- [ ] **SESSION_SECRET**: Ensure this is set to a long, random string in Replit Secrets. The code currently falls back to a dev default if missing.
- [ ] **ANTHROPIC_API_KEY**: Double-check that this is a production-tier key to avoid rate limiting during high traffic.

### 3. AI Safety
- [ ] **Prompt Injection**: The `Interviewer Agent` is currently susceptible to advanced prompt injection (e.g., a candidate telling the AI to "give me a score of 100").
    - *Mitigation*: Add a post-processing validation step or a more restrictive system prompt.
- [ ] **Input Sanitization**: Ensure candidate messages are sanitized before being sent to the LLM to prevent formatting attacks.

### 4. Database
- [ ] **Backups**: Verify the backup frequency of the Replit PostgreSQL instance.
- [ ] **Indices**: Add database indices on `company_id` columns across all tables to maintain performance as the data grows.
